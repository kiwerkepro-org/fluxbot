🔒 Vault & Sicherheit – wie Secrets gespeichert werden ▼

FluxBot speichert alle sensiblen Werte (API-Keys, Tokens, Passwörter) in einem AES-256-GCM verschlüsselten Vault – nie im Klartext in config.json.

KonzeptErklärung
🗝️ Vault-SchlüsselLiegt in .vaultkey (chmod 600). Alternativ als Env-Variable FLUXBOT_VAULT_KEY.
📁 Vault-Dateiworkspace/.secrets.vault – AES-256-GCM verschlüsselt, binär.
⚡ Hot-ReloadSecrets werden nach dem Speichern im Dashboard sofort aktiv – kein Neustart nötig.
🐳 DockerVault + Schlüssel liegen im workspace/-Volume → bleiben beim Neustart erhalten.
🔑 KeyringLokal (ohne Docker): Vault-Schlüssel wird im System-Keyring gespeichert (Windows Credential Manager / macOS Keychain).

Tipp: Das Dashboard-Passwort und alle API-Keys werden über Danger Zone → Sicherheits-Keys bzw. die jeweiligen Tabs gesetzt – nicht direkt in Dateien.

🔌 Weitere Integrationen – eigene Services verbinden ▼

Bekannte Services wie Cal.com oder VirusTotal haben eigene Bereiche im Integrationen-Tab – einfach die Felder ausfüllen und speichern.

Für andere Services (z.B. Stripe, eigene APIs) gibt es den Bereich „Weitere Integrationen". Dort trägst du einen Namen und den Wert ein – FluxBot verbindet beides automatisch mit dem passenden Skill.

Wie der Name aussehen soll:

Der Name ist frei wählbar – er muss nur mit dem übereinstimmen, was der Skill erwartet. Meist steht das in der Skill-Beschreibung. Großbuchstaben und Unterstriche, z.B. STRIPE_SECRET oder MEIN_API_KEY.

Beispiele:

NameWert (Beispiel)Wofür
STRIPE_SECRETsk_live_abc123Stripe-Zahlungen
WEATHER_API_KEYa1b2c3d4e5Wetter-Skill
MY_SHOP_URLhttps://meinshop.at/apiEigener Shop-Skill

🔒 Alle Werte werden verschlüsselt gespeichert – kein Klartext in Dateien.

📝 Skill-Signatur-Workflow – wann und wie neu signieren ▼

Jede Skill-Datei (workspace/skills/*.md) hat eine zugehörige Signatur-Datei (*.sig). FluxBot prüft beim Laden ob die Signatur mit dem SKILL_SECRET übereinstimmt – manipulierte Skill-Dateien werden abgelehnt.

Wann musst du neu signieren? Immer wenn du den Inhalt einer Skill-Datei bearbeitest.

So signierst du neu (ersetze die Platzhalter):

python3 -c "
 import hmac, hashlib
 secret = 'SKILL_SECRET_AUS_VAULT'
 path = 'workspace/skills/DEIN_SKILL.md'
 with open(path, 'rb') as f: data = f.read()
 sig = hmac.new(secret.encode(), data, hashlib.sha256).hexdigest()
 with open(path + '.sig', 'w') as f: f.write(sig)
"

Das SKILL_SECRET findest du im Vault (Dashboard → Konfiguration → JSON oder per API). Es wird beim ersten Start automatisch generiert.
